# 🚀 IMI (Integrated Merge Interface)
### The Ultimate AI Orchestration & Cloud-Sync Hub

IMI is a high-performance, Electron-based command center designed to bridge the gap between high-level AI strategy and autonomous code implementation. By separating the **"Brain" (Strategy)** from the **"Coder" (Execution)**, IMI provides a surgical, efficient, and cost-effective development workflow.

---

## 🌟 Key Features

### 1. **Hybrid Orchestration Pipeline**
IMI uses a unique dual-model architecture:
*   **The Brain:** Uses top-tier models (Gemini, GPT-4o, Claude 3.5) to analyze requests and generate precise implementation plans.
*   **The Coder:** Specialized implementation engines (Jules, Gemini-as-Coder) that receive plans and execute file system changes autonomously.

### 2. **Global MCP Hub**
A centralized registry for the **Model Context Protocol**. Link specialized power tools with a single click:
*   **Jules Agent:** Professional-grade implementation engine.
*   **GitHub Sync:** Bidirectional cloud repository access.
*   **Filesystem:** Real-time local directory monitoring.
*   **Memory:** Persistent knowledge graph for long-term project context.

### 3. **Bidirectional Cloud-Sync**
IMI keeps your local desktop and GitHub repository in perfect harmony:
*   **60-Second Heartbeat:** Silent background synchronization every minute.
*   **Instant Post-Action Push:** Automatically commits and pushes changes the moment an AI implementation finishes.
*   **Rebase-Ready Pulls:** Intelligently integrates cloud changes without messy merge conflicts.

### 4. **Multi-Model Fleet**
Support for the world's most advanced AI models via direct API streaming:
*   **Google:** Gemini 1.5 Flash & Pro.
*   **OpenAI:** ChatGPT (GPT-4o).
*   **Anthropic:** Claude 3.5 Sonnet.
*   **Open Source:** Llama 3 (via Groq), DeepSeek, and Mistral Large.

### 5. **Surgical Edit Mode**
Optimized for speed and token efficiency. IMI instructs models to perform **Surgical Edits**, meaning only the specific requested lines are modified, preventing unnecessary file rewrites and reducing latency.

---

## 🛠️ System Telemetry
*   **Real-Time CPU/RAM Monitoring:** Track system load directly from the dashboard.
*   **Token Usage Tracking:** Keep an eye on your API consumption per model.
*   **Context Snapshots:** Save and load your entire workspace state to resume deep-work sessions instantly.

---

## 🚀 Getting Started

1.  **Launch:** Open the IMI app using the `IMI_Launcher.bat` on your desktop.
2.  **Configure:** Go to the **System** tab and enter your API keys.
3.  **Link:** Visit the **Global MCP** tab and link your preferred implementation tools.
4.  **Deploy:** Go to the **Command Center** and start building with the Architect-to-Coder loop.
