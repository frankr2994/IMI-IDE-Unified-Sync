*   **IPC Handlers for Saving/Loading:**
    *   Implement `ipcMain.handle` listeners for `'save-ai-settings'` and `'load-ai-settings'`.

    *   The `load-ai-settings` handler will return the currently saved settings.
*   **Persistence Mechanism:**
    *   Use Node.js `fs` module to read from and write to a configuration file (e.g., `ai-settings.json`). This file should be stored in the user's application data directory, accessible via `app.getPath('userData')`.
    *   Define a path for the settings file:
    ```javascript
    const path = require('node:path');
    const fs = require('node:fs/promises'); // Use promises for async file operations

    const aiSettingsPath = path.join(app.getPath('userData'), 'ai-settings.json');
    ```
    *   **`save-ai-settings` handler:**
        ```javascript
        ipcMain.handle('save-ai-settings', async (event, settings) => {
            try {
                await fs.writeFile(aiSettingsPath, JSON.stringify(settings, null, 2));
                return { success: true };
            } catch (error) {
                console.error('Failed to save AI settings:', error);
                return { success: false, error: error.message };
            }
        });
        ```
    *   **`load-ai-settings` handler:**
        ```javascript
        ipcMain.handle('load-ai-settings', async () => {
            try {
                const data = await fs.readFile(aiSettingsPath, 'utf8');
                return JSON.parse(data);
            } catch (error) {
                if (error.code === 'ENOENT') {
                    // File not found, return default settings
                    return {
                        temperature: 0.7,
                        maxTokens: 2048,
                        modelId: 'gpt-4-turbo-preview',
                    };
                }
                console.error('Failed to load AI settings:', error);
                return null; // Or throw error, or return default
            }
        });
        ```
